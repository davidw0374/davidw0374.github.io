老电视盒变身下载宝（1/3）

就喜欢折腾

2010年那时候Linux电视盒子比较流行，买了一个当时大品牌开博尔的K355i，效果放在当时也还算可以。逾两年，都是流行安卓盒子了，安卓盒子天然的APP多，所以这个开博尔盒子就放在那里积灰了。现在业余时间比较多，经常要下载一点东西，挂机时间长，用电脑功耗大不环保， 想到了要把这个开博尔盒子折腾改造一下。

盒子软硬件现状

硬件

外观



接口还是比较齐全的：2*USB2.0/1*SATA/1*Ethernet 100M/1*wireless LAN

母版

  1. SoC RTD1185
  2. 256MB RAM from NANYA
  3. 256MB NAND SLC - flash Samsung
  4. 2x 2.0 USB Host
  5. FastEthernet RJ45 Socket
  6. HDMI Transmitter out
  7. IR receiver for remote control (remote control available, photo below)
  8. Lock button to turn on / off
  9. USB WiFi based on RTL8190U inserted in one USB Host
  10. Pin plug (UART)
  11. Recovery button

    CPU

  • Realtek的RTD1185使用50纳米制程，在10年前算比较先进的制程；
  • MIPS32 24Kc内核

软件

原厂固件没啥意思，刷的是绰号‘3099’牛人制作的发烧友固件，兼顾音视频播放和transmission下载。

  • Linux Venus 2.6.12.6-VENUS #13 Wed Nov 28 21:47:36 HKT 2012 mips unknown
  • BusyBox 1.1.3(2012.11.28-13:48+0000) Built-in shell (ash)
  • audio&video app （ I don’t care）
  • samba v2
  • Transmission 2.92

改造可行性分析

劣项

  • SOC RTD1185
    • MIPS体系架构大势已去。同为RISC体系，十几年竞争下来，ARM胜者为王，MIPS苟延残喘。
    • 主频500Mhz
  • linux 内核2.6.12.6，2008年发布的内核，版本确实比较低了
  • 256M RAM

劣势比较明显，不过做专业下载机也基本够用。

优势

  • 好不容易拿到了SDK

年代久远的老机器了，网络上好多相关的内容已经删除。为了下载SDK，竟然还上了一次钓鱼网站的当，一个国外网站伪装成下载论坛，有楼主帖还有搞配合的帖子，挺像那么一回事，结果导向一个网站索要信用卡信息提供下载，我竟然鬼迷心窍发送了，发送以后打开下载链接发现是死链，马上感觉掉坑了， 赶紧电话去信用卡中心注销卡服务费40元。
网上又搜寻了一圈，无影无踪，偶尔后来看资料说1073和1185的SDK是一样的，网上七上八下总算找到了1073的SDK，哇塞真不容易。

  • Ubuntu用得比较顺溜了

用电视盒子改造成下载机，主要就是做软件的减法了，杀伐决断，更改配置，调试运行，都需要在Ubuntu远程主机上操作

工作思路

  1. 编译交叉工具链
  2. 编译内核，使之支持NFS，便于远程主机调试
  3. Ubuntu和box配合调试
    1. 删除所有与下载无关的软件
    2. 更新busybox版本，samba版本
    3. 更新transmission版本，新增aria2下载软件
  4. 封装打包成U盘加载文件
  5. 通过U盘加载刷机
